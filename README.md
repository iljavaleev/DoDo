# DoDo
``` 
Программа получает данные от пользователя из коммандной строки:
    --video - список имен файлов
    --output - путь, куда сохранить видео

Обрабатывает данные:
    - останавливается для выбора пользователем фрейма
    - с помощью нейросети определяет два класса во фрейме: 
    персона и стол. Единожды определив координаты стола, больше их не 
    меняем, чтобы изображение было более стабильным. 
    Тем более, что объект статичный
    - используем координаты bbox от нейросети для формирования 
    статуса столика и рисуем соответсвующнго цвета bbox
        -- стол занят - есть персона и стол, координаты пересекаются
        (Зеленый)
        -- человек около стола - координаты не пересекаюся
        (Красный)
        -- стол пуст - персона отсутсвует или не определена
        (Синий)
    - пишем обработанные фреймы в файл
    - записываем временные метки для каждого статуса и далее формируем датафрейм


Анализирует данные:
    - Так как нужно определить "среднее время между уходом гостя и подходом 
    следующего человека", то можно приравнять 'Столик занят' и 
    'человек около стола'. Здесь по разному можно трактовать; 
    мы не сможем определить какой человек у стола: тот же или другой 
    посетитель. В любом случае статус 'человек около стола' будет лишним.
    - Находим ряды с изменением статуса
    - Находим разницу между временными метками измений статуса
    - Так как статус привязан к работе нейросети, то получается большое 
    количество шума(ложных срабатываний). Уменьшаем количество шума, выбирая 
    ряды с разницей больше секунды
    - Ряды со статусом 0 - это событие, когда столик стал занят после того 
    как он был свободен. И разница указывает на количество времени 
    простаивания. Вычисляем среднее время простаивания

Отчет:
    Я использовал yolov8n, и она хороша для определения персон, но столы вот 
    не очень хорошо видит. В моей реализации определение статуса жестко 
    привязано к координатам bbox для стола, поэтому чем быстрее сеть определит 
    стол, тем раньше начнется запись событий. При этом чем больше фрейм 
    выбираешь тем меньше вероятность, что он будет определен. Отсюда сложности с
    тем чтобы определить человека около стола, потому что он не попадает во фрейм. 
    Плюс столы, если расположены под углами и т.д., сеть не определяет. 
    Еще одна сложность - фиксация статусов: сеть определяет покадрово и иногда 
    не срабатывает, персона может выпасть к примеру, и получается ложное 
    срабатывание изменения, что засоряет данные. А эти скачки очень пагубно 
    влияют на средний результат.
    Выбрал второе видео, там к сожалению мало событий, но хороший обзор, и стол 
    сразу определяется.
Результат 
    Среднее время простоя столика: 34.51 секунд.
    Хотя там видно, что намного больше. Но из-за скачков среднее время заметно 
    уменьшается. Возможно, нужно другую технику для анализ применить

    Ссылка на видео на всякий случай:
    https://disk.yandex.ru/i/dGVQExBoxHlsLg
```
## Установка и использование

```bash
git clone https://github.com/iljavaleev/DoDo.git

cd DoDo 
python -m venv venv && source venv/bin/activate
pip install .
main --video samples/video2.mp4 --output output.mp4
```
